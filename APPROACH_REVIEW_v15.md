# Frequency Estimation for Periodic and Rhythmic EEG Patterns: Review v15

## Critical Methodological Principle

**No method may use gold standard labels as input.** All algorithms operate from raw EEG only.

## Dataset (as of 2026-03-25)

### EEG Data

11,817 standardized .mat files (18 channels × 2000 samples at 200 Hz).

The raw per-segment votes are in `data/labels/archive_labels/list_events_20241129.csv`. Patient-level vote aggregates (previously in patients.csv) are misleading — they sum votes across all segments for a patient, mixing LRDA segments with "other" segments from the same patient.

## Unified PD Characterization Pipeline (PDCharacterizer)

A single per-channel CNN (**ChannelPD-Net**) serves as the backbone for all PD analysis tasks, feeding into three downstream modules:

```
EEG (18ch × 2000 @ 200Hz)
│
├── ChannelPD-Net (×18 channels, 5-fold CNN+Attention)
│   → 18 PD probabilities + 18 frequency estimates
│   │
│   ├── Laterality Detection (LPD only)
│   │   mean(left probs) vs mean(right probs) → L/R
│   │   AUC = 0.98 (437 cases)
│   │
│   ├── Hybrid-PLV Spatial Localizer
│   │   Top-3 CNN channels → reference phase
│   │   PLV to each channel → region scores
│   │   Composite = 0.811, AUC = 0.814 (465 segments)
│   │
│   └── CNN-Weighted HemiCET+DP
│       CNN probs weight channel evidence aggregation
│       HPP + CET → product-boost → DP + EM
│       F1 = 0.684 (651 patients)
│
Output: laterality, regions, discharge_times, frequency
```

**Key insight**: ChannelPD-Net serves triple duty — laterality (which hemisphere), localization (which regions via PLV reference), and timing (which channels to weight). Laterality detection feeds forward into both downstream modules: it constrains the spatial localizer to seed from the ipsilateral hemisphere and restricts the discharge detector to ipsilateral channels. This eliminates redundant computation and ensures consistency across tasks.

**Hybrid-PLV** outperforms both pure PLV (composite 0.789) and pure CNN (0.787) by using CNN-informed reference channel selection with physics-based phase coherence analysis. The CNN identifies the PD epicenter; PLV identifies connected regions.

**Code**: `code/pd_characterizer.py` — standalone callable pipeline.

**Full architecture**: See [paper_materials/unified_pd_pipeline.md](paper_materials/unified_pd_pipeline.md).

## RDA Analysis Contest

45 methods evaluated across 3 tasks on 1,267 RDA segments (475 LRDA + 792 GRDA):

| Task | Best Method | Score |
|------|-------------|-------|
| Expert agreement prediction | Ensemble Ridge (18 features) | ρ = 0.235 |
| RDA detection (vs "other") | Ensemble Ridge | AUC = 0.634 |
| Frequency estimation | M3_HilbertCV | ρ = 0.593 |

**M3_HilbertCV**: Instantaneous frequency via Hilbert transform on delta-filtered (0.5-3.5 Hz) signal. CV of instantaneous frequency measures rhythmic regularity — low CV = steady rhythm = clear RDA.

**Expert agreement** remains difficult to predict (ρ=0.235). Signal-level features from a single 10-second segment explain ~5% of variance in expert agreement. The remaining variance likely reflects clinical context, artifact assessment, and pattern evolution.

## What Changed Since v14

### 0. LRDA vs GRDA Classification (NEW — Task 2)

Built a multi-round contest to discriminate lateralized from generalized RDA.

#### Dataset
- IIIC crowd-labeled cases: ≥10 expert votes, ≥50% agreement on LRDA or GRDA
- After MW expert review of 172 disagreement cases: 80 labels corrected (72 LRDA→GRDA, 8 GRDA→LRDA)
- Final corrected dataset: 247 LRDA + 569 GRDA (balanced to 247+249 for training)

#### Progression (4 rounds)

| Round | Approach | Best AUC | Key Insight |
|-------|----------|----------|-------------|
| V1 | Hemisphere-independent scoring (25 methods) | 0.58 | Simple L/R asymmetry is weak |
| V2 | ML on basic features (126 combos) | 0.753 | Per-channel features help |
| V3 | Comprehensive features (69 combos, 9 families) | 0.796→0.819 | Spatial focality dominates |
| V3+ | Expert label correction + re-eval | **0.870** | Cleaner labels = +0.05 AUC |

#### Winning Method: spatial_all + GBM (AUC = 0.870)

124 spatial features from 4 families, classified by Gradient Boosted Trees (200 trees, depth 4):

**Feature families ranked by individual AUC (with RF):**

| Family | AUC | Features | What it captures |
|--------|-----|----------|-----------------|
| Focality | 0.720 | 13 | Spatial concentration (Gini, entropy), hemisphere imbalance |
| Homologous pairs | 0.719 | 52 | PLV, correlation, power ratio between L↔R channel pairs |
| Per-channel | 0.714 | 54 | Power, spectral peak ratio, Hilbert Q per channel |
| Background sub | 0.615 | 5 | Homologous pair residual power/rhythmicity |

**Key finding**: LRDA vs GRDA is fundamentally about **focal vs diffuse** spatial organization, not just amplitude asymmetry. The top features are: spatial Gini coefficient, homologous pair PLV breaks, per-channel power concentration, and hemispheric imbalance.

**Bandpass optimization**: 0.5–5.0 Hz slightly outperforms 0.5–4.0 Hz on uncorrected labels (0.819 vs 0.797), but with corrected labels the original 0.5–4.0 Hz band is best (0.870).

#### Next: LRDA side labeling
With LRDA vs GRDA discrimination at AUC=0.870, the next step is labeling LRDA cases as left or right dominant to solve the full lateralization problem.

#### Key Findings

1. **Cross-channel methods dominate**: The top 2 methods (phase coherence, cross-correlation) measure relationships between channels, not per-channel features. This makes clinical sense — PDs propagate through connected brain regions, so the connectivity pattern is more informative than isolated channel features.

2. **The "all regions" baseline is hard to beat**: With 66-84% base rate per region, predicting all regions involved yields F1=0.847 and Jaccard=0.736. The real discrimination challenge is identifying the ~20-30% of regions NOT involved.

3. **Subtype-aware baseline is surprisingly strong**: Simply predicting "all regions" for GPD and "ipsilateral 4 regions" for LPD (B2_SubtypeDefault, composite=0.769) outperforms most signal-processing methods. It also has the best spatial extent correlation (ρ=0.786). Subtype explains much of the spatial pattern.

4. **AUC is the discriminating metric**: Most methods achieve similar F1 (0.84-0.86) but differ substantially on AUC (0.50-0.72). The ability to produce calibrated continuous scores separating involved from uninvolved regions is what separates winners from baselines.

5. **CNN models failed to load**: The M1-M4 methods (using pretrained ChannelPDNetAttention) fell back to power-based scoring because model weights were not in the expected paths. These should be re-run after fixing model paths — they are likely competitive.

6. **Gradient sharpness has best extent correlation** (ρ=0.614) among non-baseline methods, suggesting kurtosis of the signal derivative captures something about how widespread the discharges are.

#### Methods Tested

**Baselines (4):** All-regions, subtype-default (GPD=all/LPD=ipsilateral), RMS threshold, random.

**Signal-based (8):** Pointiness max/mean, FFT peak power, bandpower ratio, ACF peak height, line length, envelope peakiness, spectral entropy.

**Cross-channel (6):** Coherence network, cross-correlation with best channel, peak synchrony, template correlation, phase-locking value, mutual information.

**Hybrid (8):** Ensemble vote, weighted ensemble, adaptive threshold, power rank percentile, symmetry-aware, TKEO energy, per-channel VE sweep, gradient kurtosis.

**CNN model (4):** Pretrained ChannelPDNet probabilities, attention-weighted, CNN+pointiness blend, CNN with GPD symmetry enforcement (all failed due to model path issue).

### 2. RDA Wave Auto-Labeling Pipeline

Built automated per-wave timing annotation for 549 RDA cases with expert-verified frequency:

**Algorithm:**
1. Load 18-channel bipolar EEG
2. Detrend, then bandpass filter around known frequency (f ± adaptive bandwidth)
3. Average filtered signal across involved channels per hemisphere
4. Find peaks (local maxima) = wave peaks
5. Find zero crossings = wave onset/offset times
6. Compute quality metrics (IPI coefficient of variation, peak height consistency)
7. Assign review tier based on quality

**Three-tier quality system:**
- **Tier 1 (auto-accept)**: IPI CV < 0.20 — clean, regular waves
- **Tier 2 (quick-review)**: IPI CV 0.20-0.35 — mostly regular, may have minor issues
- **Tier 3 (manual-edit)**: IPI CV > 0.35 — irregular, needs expert correction

**Interactive HTML review tool** built with:
- EEG canvas display with channel grouping (L-lateral, L-parasag, midline, R-parasag, R-lateral)
- Bandpass-filtered overlay showing detected wave pattern
- Red vertical lines at detected wave peaks
- Blue shaded regions showing wave onset-to-offset intervals
- Add/delete mode for correcting wave peaks (click to add, click to delete)
- Accept/reject buttons with keyboard shortcuts
- Auto-advance, progress tracking, localStorage persistence, JSON export

### 3. RDA HemiCET+DP Design

Wrote design document (`code/rda_detector/DESIGN_rda_hemicet.md`) planning the adaptation of the PD HemiCET+DP pipeline for LRDA/GRDA detection:

**What transfers directly:**
- Dynamic programming (periodicity prior, skip tolerance, Viterbi backtracking)
- Pipeline architecture (evidence → threshold → candidates → DP → template refinement)
- Hemisphere-based processing
- Frequency = 1/median(IPI)

**What needs adaptation:**
- Evidence trace: wider Gaussian targets (σ=25-50ms vs 10ms) for rounder RDA waveforms
- Candidate peak extraction: smoothed peaks rather than sharp transients
- DP parameters: lower timing penalty (RDA periods more variable)
- Baseline option: bandpass envelope as evidence (no neural model needed)

**Training data path:**
- 549 cases with auto-labeled wave timings (from Step 2 above)
- Three-tier review for quality control
- Expert review → cleaned training set → retrain "RhythmiCET" U-Net

## Architecture

### PD Detection: HemiCET+DP (Winning Model, F1=0.891)

```
INPUT: 8 hemisphere channels (4 temporal + 4 parasagittal)
│
├── BRANCH A: Frequency Estimation
│   ├── CNN+Attention (×8 ch, ×5 fold) → p_ch, f_ch
│   ├── PD-weighted average → f_CNN
│   ├── Per-channel ACF → f_ACF
│   └── f_est = clip(0.8×f_CNN + 0.2×f_ACF, 0.3, 3.5)
│
├── BRANCH B: Evidence Generation
│   └── HemiCET U-Net (8→1, 525K params, 5-fold) → E(t) ∈ [0,1]
│
├── Evidence thresholding (50th percentile)
├── Candidate peak extraction
├── Dynamic Programming (α=1.5, β=0.3, λ=0.05, max_skip=3)
├── EM template refinement (average→correlate→re-DP)
├── Post-hoc confidence filter (< 0.4× median)
│
└── OUTPUT: discharge times + frequency (1/median IPI)
```

### PD Spatial Localization (Best: Phase Coherence, Composite=0.789)

```
INPUT: 18-channel bipolar EEG
│
├── Find reference channel (highest PD-band power)
├── Compute phase-locking value (PLV) between reference and each channel
├── Map 18 per-channel PLV scores → 8 brain regions (max per region)
├── Threshold → involved regions
│
└── OUTPUT: {LF, RF, LT, RT, LCP, RCP, LO, RO} involvement + spatial extent
```

### BIPD Detection: Two-Stage Pipeline

```
EEG (18ch) ──┬── Left 8ch  ──→ HemiCET+DP ──→ {t_L}, f_L ──┐
             └── Right 8ch ──→ HemiCET+DP ──→ {t_R}, f_R ──┤──→ 21 features ──→ GBT ──→ P(BIPD)
```

### RDA Frequency: NVO + Wave Timing (Planned)

```
EEG (18ch) ──→ bandpass at f±bw ──→ detrend ──→ peak detection ──→ wave times ──→ freq
                                                                  ──→ HemiCET-RDA (future)
```
